El Problema de la Mochila

Optimización combinatoria clásica

¿Qué es la Optimización Combinatoria?

Buscar la mejor solución dentro de un conjunto finito pero enorme de posibilidades.

  • Finito: las variables son discretas (enteras o binarias)
  • Enorme: con $n$ ítems binarios hay $2^n$ combinaciones
  • NP-hard: no se conoce algoritmo eficiente para el caso general
Con solo 5 ítems: $2^5 = 32$ combinaciones. Con 100 ítems: $2^{100} \approx 10^{30}$ 🤯

El Problema de la Mochila (Knapsack)

Tienes una mochila con capacidad limitada. Cada ítem tiene un valor y un peso. ¿Cuáles seleccionas para maximizar el valor sin exceder la capacidad?

$$\max \sum_{i=1}^{n} v_i \, x_i \qquad \text{sujeto a} \qquad \sum_{i=1}^{n} w_i \, x_i \leq W$$

Donde $x_i \in \{0, 1\}$: seleccionar (1) o no (0) el ítem $i$.

Del ILP al QUBO

Transformación paso a paso

¿Por qué necesitamos QUBO?

ILP (clásico)

  • Función objetivo + restricciones separadas
  • Los solvers clásicos manejan restricciones directamente

QUBO (cuántico)

  • Sin restricciones — todo en una sola función
  • Mapeable a Hamiltoniano Ising
  • Compatible con QAOA y Quantum Annealing
QUBO = Quadratic Unconstrained Binary Optimization
Las restricciones se incorporan como penalizaciones cuadráticas.

Paso 1: Convertir a Minimización

Los computadores cuánticos buscan el estado de mínima energía:

$$\max \sum v_i x_i \;\;\longrightarrow\;\; \min \left(-\sum_{i=1}^{5} v_i x_i\right)$$

Para nuestra instancia:

$$\min \;\; -(8x_1 + 10x_2 + 15x_3 + 4x_4 + 12x_5)$$

Paso 2: Penalización de Restricciones

Incorporamos la restricción como un término de penalización cuadrática:

$$\min \;\; \underbrace{-\sum_{i} v_i x_i}_{\text{objetivo}} \;+\; \lambda \underbrace{\left(\sum_{i} w_i x_i - W\right)^2}_{\text{penalización}}$$
  • $\lambda > 0$: fuerza de penalización (debe ser suficientemente grande)
  • Si la restricción se cumple: penalización $= 0$
  • Si la restricción se viola: penalización $> 0$ (castigo)
Regla práctica: $\lambda > \max(v_i)$. En nuestro caso: $\lambda = 16$

Paso 3: Variables de Holgura (Slack)

Problema: la restricción es de desigualdad ($\leq$), pero necesitamos igualdad.

$$\sum_{i} w_i x_i \leq W \;\;\longrightarrow\;\; \sum_{i} w_i x_i + s = W$$

Donde $s \geq 0$ es la variable de holgura (slack) que absorbe la diferencia.

  • Si peso $= 15$ (llena): $s = 0$
  • Si peso $= 12$: $s = 3$ (sobran 3 unidades)
  • Rango de $s$: $\{0, 1, 2, \ldots, 15\}$

Paso 4: Expansión Binaria del Slack

Para que $s$ sea compatible con variables binarias, la representamos con 4 bits:

$$s = 1 \cdot s_0 + 2 \cdot s_1 + 4 \cdot s_2 + 8 \cdot s_3 \qquad s_k \in \{0, 1\}$$

Esto cubre $s \in \{0, 1, 2, \ldots, 15\}$ (necesitamos $\lceil \log_2(15+1) \rceil = 4$ bits).

Paso 6: Forma QUBO Final

Juntando todo — objetivo + penalización con slack expandido:

$$C(\mathbf{x}, \mathbf{s}) = -\sum_{i=1}^{5} v_i x_i + \lambda \left(\sum_{i=1}^{5} w_i x_i + \sum_{k=0}^{3} 2^k s_k - 15\right)^{\!2}$$

Expandiendo el cuadrado y usando $x_i^2 = x_i$ (propiedad binaria):

$$C = \sum_{j} Q_{jj} \, y_j + \sum_{j < k} Q_{jk} \, y_j y_k + \text{constante}$$

Donde $\mathbf{y} = (x_1, \ldots, x_5, s_0, \ldots, s_3)$ y $Q$ es la matriz QUBO.

Del QUBO al Hamiltoniano Ising & QAOA

Mapeo cuántico y algoritmo variacional

Mapeo QUBO → Ising

En un computador cuántico, las variables binarias se codifican con qubits usando operadores de Pauli $Z$ (actuando sobre el estado de cada qubit):

$$\hat{y}_j = \frac{I - Z_j}{2} \qquad \text{donde } Z_j = \begin{pmatrix} 1 & 0 \\ 0 & -1 \end{pmatrix}$$

Donde $I$ es el operador Identidad. Esto nos da los eigenvalores:

Estado cuánticoEigenvalor de $Z_j$Valor de $\hat{y}_j$ (medida)Significado físico
$|0\rangle$$+1$$\frac{1 - 1}{2} = 0$No seleccionado / Slack inactivo
$|1\rangle$$-1$$\frac{1 - (-1)}{2} = 1$Seleccionado / Slack activo

Desarrollo del Hamiltoniano $H_C$ — Sustitución

Sustituimos el operador $\hat{y}_j = \frac{I - Z_j}{2}$ en la función de costo QUBO:

$$H_C = \sum_{j} Q_{jj} \left(\frac{I - Z_j}{2}\right) + \sum_{j < k} Q_{jk} \left(\frac{I - Z_j}{2}\right)\left(\frac{I - Z_k}{2}\right) + C_{\text{offset}} I$$

Expandiendo el término cuadrático $\left(\frac{I - Z_j - Z_k + Z_j Z_k}{4}\right)$ y agrupando en base a los operadores de Pauli $Z$:

$$H_C = \sum_{j} h_j Z_j + \sum_{j < k} J_{jk} Z_j Z_k + C'_0 I$$

Coeficientes del Hamiltoniano Ising

Las relaciones exactas entre la matriz QUBO $Q$ y los coeficientes del Hamiltoniano $H_C$ son:

$$h_j = -\frac{Q_{jj}}{2} - \frac{1}{4}\sum_{k \neq j} Q_{jk}$$
$$J_{jk} = \frac{Q_{jk}}{4}$$

* $h_j$: campos locales (qubit bias).
* $J_{jk}$: acoplamientos entre qubits (qubit-qubit interaction).

Cálculo del Qubit 1 ($x_1$)

Apliquemos las fórmulas al primer qubit ($x_1$) de nuestra instancia de mochila:

  • Término diagonal: $Q_{11} = -1304$
  • Términos cruzados: $Q_{12}=480, Q_{13}=672, Q_{14}=192, Q_{15}=576$, $Q_{16}=96, Q_{17}=192, Q_{18}=384, Q_{19}=768$ (Suma = $3360$)

Calculamos su campo local $h_1$ y su acoplamiento con el Qubit 2 ($J_{12}$):

$$h_1 = -\frac{-1304}{2} - \frac{3360}{4} = 652 - 840 = \mathbf{-188}$$ $$J_{12} = \frac{480}{4} = \mathbf{120}$$

De $H_C$ al Operador Unitario $U_C(\gamma)$

El operador de evolución de costo se formula como la exponencial del Hamiltoniano:

$$U_C(\gamma) = e^{-i \gamma H_C} = \prod_{j} e^{-i \gamma h_j Z_j} \prod_{j < k} e^{-i \gamma J_{jk} Z_j Z_k}$$

Esto se descompone en compuertas cuánticas elementales de rotación:

Términos Lineales: $e^{-i \gamma h_j Z_j}$

Se implementa directamente con una compuerta de rotación en el eje Z:

$R_z(2\gamma h_j)$

Términos Cuadráticos: $e^{-i \gamma J_{jk} Z_j Z_k}$

Requiere entrelazamiento usando dos compuertas CNOT y una rotación Z:

$\text{CNOT}_{j,k} \to R_z(2\gamma J_{jk})_k \to \text{CNOT}_{j,k}$

¿Qué es QAOA?

Quantum Approximate Optimization Algorithm

Algoritmo variacional híbrido (cuántico-clásico) propuesto por Farhi et al. (2014)

Parte Cuántica

  • Prepara un estado con un circuito parametrizado
  • Mide el estado resultante

Parte Clásica

  • Evalúa la función de costo
  • Optimiza los parámetros del circuito

El Circuito QAOA

Se aplican $p$ capas alternas de dos operadores unitarios:

$$|\boldsymbol{\gamma}, \boldsymbol{\beta}\rangle = \prod_{l=1}^{p} \; e^{-i\beta_l H_M} \; e^{-i\gamma_l H_C} \; |+\rangle^{\otimes n}$$

$e^{-i\gamma H_C}$

Operador de costo: imprime la estructura del problema en las fases de los qubits

$e^{-i\beta H_M}$

Operador de mezcla: $H_M = \sum_i X_i$. Explora el espacio de soluciones

Parámetros: $2p$ valores ($\gamma_1,\ldots,\gamma_p, \beta_1,\ldots,\beta_p$) optimizados clásicamente

Bucle de Optimización Variacional

1
Inicializar: Estado uniforme $|+\rangle^{\otimes n}$ + parámetros aleatorios $\gamma, \beta$
2
Circuito cuántico: Aplicar $p$ capas de $U_C(\gamma) \cdot U_M(\beta)$
3
Medir: Obtener cadenas de bits y calcular $\langle H_C \rangle$
4
Optimizar: COBYLA ajusta $\gamma, \beta$ para minimizar $\langle H_C \rangle$
5
Repetir pasos 2-4 hasta convergencia
Más capas ($p$ mayor) → mejor aproximación, pero circuito más profundo y costoso.
04

Quantum Annealing

Optimización por evolución adiabática

¿Qué es Quantum Annealing?

Paradigma de computación cuántica basado en el Teorema Adiabático: si un sistema cuántico evoluciona suficientemente lento, permanece en su estado de mínima energía.

QAOA (compuertas)

  • Circuito discreto de compuertas
  • $p$ capas parametrizadas
  • IBM Quantum / Qiskit

Annealing (continuo)

  • Evolución continua del Hamiltoniano
  • Sin compuertas ni capas
  • D-Wave Systems

Evolución del Hamiltoniano

$$H(t) = \underbrace{\left(1 - \frac{t}{T}\right) H_{\text{init}}}_{{\text{fácil: superposición}}} + \underbrace{\frac{t}{T} \; H_{\text{final}}}_{{\text{problema: QUBO/Ising}}}$$
1
$t = 0$: $H(0) = H_{\text{init}}$ — Estado base fácil de preparar (superposición uniforme)
2
$0 < t < T$: Evolución lenta — el sistema sigue en su estado base
3
$t = T$: $H(T) = H_{\text{final}} = H_C$ — El estado base codifica la solución óptima

Notebook 01 Solver Clásico — PuLP

import pulp

# Crear modelo de maximización
modelo = pulp.LpProblem("Mochila", pulp.LpMaximize)

# Variables binarias: x_i ∈ {0, 1}
x = [pulp.LpVariable(f"x_{i}", cat='Binary') for i in range(5)]

# Objetivo: max 8x₁ + 10x₂ + 15x₃ + 4x₄ + 12x₅
modelo += pulp.lpSum([valores[i] * x[i] for i in range(5)])

# Restricción: 3x₁ + 5x₂ + 7x₃ + 2x₄ + 6x₅ ≤ 15
modelo += pulp.lpSum([pesos[i] * x[i] for i in range(5)]) <= 15

# Resolver con CBC (gratuito, incluido)
modelo.solve(pulp.PULP_CBC_CMD(msg=0))
print(f"Valor óptimo: {pulp.value(modelo.objective)}")  # → 31

Notebook 02 QAOA — Qiskit 2.4

from qiskit_optimization.applications import Knapsack
from qiskit_optimization.algorithms import MinimumEigenOptimizer
from qiskit_algorithms import QAOA
from qiskit_algorithms.optimizers import COBYLA
from qiskit.primitives import StatevectorSampler

# Definir problema → QuadraticProgram automático
knapsack = Knapsack(values=valores, weights=pesos, max_weight=15)
qp = knapsack.to_quadratic_program()

# Configurar QAOA (p=2 capas, simulación local)
qaoa = QAOA(sampler=StatevectorSampler(),
            optimizer=COBYLA(maxiter=200), reps=2)

# Resolver
result = MinimumEigenOptimizer(qaoa).solve(qp)
print(result.prettyprint())
